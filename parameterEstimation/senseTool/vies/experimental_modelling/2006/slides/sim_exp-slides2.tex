\textbf{Definition: }\\
{\slshape
\textbf{Model:} en {\blue repræsentation} -- i en brugbar form --
{\blue af de essentielle dele} af et system.
}

\medskip
\fromSlide{2}{%
\textbf{Definition: }\\
{\slshape
\textbf{Systemidentifikation:} {\blue Udvikle matematisk model af dynamisk system
baseret på observerede data} fra systemet:
\begin{itemize}
\item Mange måledata er indsamlet som samplede værdier af input og
  output
\item En computer anvendes til behandling af data
\item Model parametre estimeres ved minimering af et fejlkriterie
\end{itemize}
}
}

\begin{slide}[Replace]{Fysiske parametre}

\vspace*{-2mm}
Fysiske parametre er {\blue model parametre med en indlysende fysisk mening}
eller betydning.

\medskip
\fromSlide{2}{%
Typisk er de {\blue koefficienter i basale fysiske love}, fx. Newtons, Hooks,
Ohms, eller Kirchoffs love.
}

\medskip
\fromSlide{3}{%
Eksempler på fysiske parametre er
\begin{itemize}
\item Mekanik parametre: Masse, friktions koeffic., stivhed
\item Elektro parametre: Modstand, induktans, kapacitet
\item Termiske parametre: Termisk modstand, specifik varme
\item Desuden: Statisk forstærkning, tidskonstant, egen-frekvens og
  dæmpningsfaktor
\end{itemize}
}

\medskip
\fromSlide{4}{%
{\red Ikke}-fysiske parametre: {\blue koefficienter i $z$-transformen} af en
tilstandsbeskrivelse.
}

\end{slide}

\overlays{4}{
\begin{slide}[Replace]{Simulering af lineært system i Matlab}
Senstools behøver en Matlab funktion: \textsf{\blue y =
  simprocess(u,t,par)}

Eksempel: Lineært system \quad $\displaystyle\blue \frac{Y(s)}{U(s)} = \frac{K}{1+s\tau}$
\begin{itemstep}
\item[\red a)] Tistandsmodel og for-løkke
\onlySlide*{1}{%

\hspace*{-5mm}%
\begin{minipage}[c]{.48\linewidth}
Kontinuert tilstandsmodel:
\blue
\begin{align*}
\dot{x}(t) & = -\frac{1}{\tau}x(t) + \frac{K}{\tau}u(t)\\
y(t) & = x(t)
\end{align*}
\end{minipage}\hspace*{5mm}
\begin{minipage}[c]{.48\linewidth}\tiny
\texttt{
\mbox{function y=simktauloop(u,t,par)}\\
% y=simktauloop(u,t,par) simulates K/(1+s*tau)
% using ss (xdot=-1/tau*x+k/tau*u, y=x) c2d and a for loop.
% par=[K tau]
%
% 22/10-02,MK
h=t(2)-t(1);\\
nn=length(u);\\
x=0;\\
K=par(1); tau=par(2);\\
{\red sysc=ss(-1/tau,K/tau,1,0);\\ 
sysd=c2d(sysc,h);\\ 
{}[ad,bd,cd,dd] = ssdata(sysd);}\\
{\blue for jj=1:nn\\
x1=ad*x + bd*u(jj);\\
y(jj)=cd*x1;\\
x=x1;}\\
end
}
\end{minipage}
}
\item[\red b)] 'filter' funktion
\onlySlide*{2}{%

\medskip
\hspace*{-5mm}%
\begin{minipage}[c]{.48\linewidth}
Kont. overføringsfunktion:
{\blue
$$ \frac{Y(s)}{U(s)} = \frac{K}{s\tau + 1} $$

\texttt{y = filter(nz,dz,u)}}

\medskip
beregner output baseret\\
 på diskret overføringsfkt.
\end{minipage}\hspace*{5mm}
\begin{minipage}[c]{.48\linewidth}\tiny
\texttt{
\mbox{function y=simktaufilt(u,t,par)}\\
% y=simktauloop(u,t,par) simulates K/(1+s*tau)
% using ss (xdot=-1/tau*x+k/tau*u, y=x) c2d and a for loop.
% par=[K tau]
%
% 22/10-02,MK
$\vdots$\\
{\red sysctf=tf(par(1),[par(2) 1]);\\
sysdtf=c2d(sysctf,h);\\{}
[nz,dz]=tfdata(sysdtf,'v');\\
\hspace*{5mm} \% NB: 'v' for vector format
\hspace*{5mm} \% - not cell}\\
{\blue y=filter(nz,dz,u);}\\
end
}
\end{minipage}
}
\item[\red c)] 'lsim' fkt.
\onlySlide*{3}{%

\hspace*{-5mm}%
\begin{minipage}[c]{.48\linewidth}
{\blue \texttt{y = lsim(sysc,u,t)}}

\medskip
beregner output baseret\\
på kontinuert system-\\ 
beskrivelse.
\end{minipage}\hspace*{5mm}
\begin{minipage}[c]{.48\linewidth}\tiny
\texttt{%
\mbox{function y=simktau(u,t,par)}\\
% y=simktauloop(u,t,par) simulates K/(1+s*tau)
% using ss (xdot=-1/tau*x+k/tau*u, y=x) c2d and a for loop.
% par=[K tau]
%
% 22/10-02,MK
$\vdots$\\
{\red nc=K; dc=[tau 1];\\
t=[0 t(1:length(t)-1)];\\
\hspace*{5mm}  \% lsim requires that t\\
\hspace*{5mm}  \% starts with 0}\\
{\blue y=lsim(nc,dc,u,t);}\\
end
}
\end{minipage}
}
\end{itemstep}

\fromSlide{4}{%
\bigskip\bigskip
Evaluering af de tre metoder:
\begin{tabular}{@{}lp{.8\linewidth}}
\red a) & er langsom, {\blue\textbf{undgå} løkker i Matlab}, hvor det
  er muligt. Ofte nødvendige ved ulineære systemer.\\
\red b) og c) & er sammenlignelige i hastighed.
\end{tabular}
}

\end{slide}
}

\overlays{3}{
\begin{slide}[Replace]{Simulering af ulineært system i Matlab}

\vspace*{-4mm}
\hspace*{\fill}\includegraphics[width=.85\linewidth]{simulating_nonlinear_sys.md}\hspace*{\fill}

Mætning: {\tiny\blue\begin{tabular}{r@{\ }c@{\ }ll}
$e$ & $<$ & $-e_0$: & $u = -u_0$ \\
$-e_0 \leq$ & $e$ & $\leq e_0$: & $u = ke$ \\
$e_0$ & $<$ & $e$: & $u = u_0$ 
\end{tabular}}

\medskip
\fromSlide{2}{%
Implementation i Matlab: \quad
\begin{minipage}[c]{4cm}\tiny\blue
\texttt{%
u = ke\\
if e>e0, u=u0; end\\
if e<-e0, u=-u0; end
}
\end{minipage}
}

\medskip
\fromSlide{3}{%
Brug af logiske operatorer (falsk udtryk har værdi 0) er mere
effektivt:\quad
{\blue\scriptsize\texttt{u = k*e-(abs(e)>e0)*k*(e-sign(e)*e0);}}

\medskip
idet $u_0 = ke_0$ kan skrives som $\blue u_0 = ke-k(e-e_0)$ og $\blue -u_0 = ke-k(e+e_0)$.
}

\end{slide}
